Definio da arquitetura de software
Atravs da utilizao do STM32F411CEU6 conseguimos
alcanar o controlo dos motores envolvidos na movimentao do
rob movel, para tal so utilizadas as funes de timers em
modo encoder para obtermos os valores de quanto os motores
se movimentam de modo a sabermos a sua posio. tambm
utilizado um timer para gerar pulsos de modo a controlar o duty
cycle do PWM.
O STM32F411CEU6 comunica com o ESP32-WROOM-32D
atravs de dois buffers TxBuffer para o envio e RxBuffer para
receber. A dimenso deste buffer definida em cdigo podendo
tomar valores arbitrrios. No final da transmisso utilizamos a
callback HAL_SPI_TxRxCpltCallback para armazenarmos os
valores recebido no buffer e escrever os novos valores no buffer
de transmisso para serem enviados na prxima comunicao.
Dentro do TxBuffer residem os valores dos counters dos
encoders de cada um dos motores. Dentro do RxBuffer
recebido o comando da prxima ao a realizar.
O ESP32-WROOM-32D o master, sendo ento este que
controla quando as comunicaes ocorrem. tambm neste que
se encontra o clock que sincroniza as comunicaes entre o
hardware.
O ESP32 usa wifi em modo "station", liga-se rede LAN
para um dispositivo acessar uma pgina Web onde interagimos
com o ESP32. A comunicao entre o ESP32 e a Web feita
atravs de "XML tables". Nesta pgina web podemos no s
obter controlo manual do rob como tambm ter acesso aos
valores dos sensores incorporados no sistema. Numa fase mais
avanada do projeto ser tambm possvel visualizar um mapa
com o percurso percorrido do rob.
